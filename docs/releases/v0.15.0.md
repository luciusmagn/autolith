# Autolith 0.15.0

Autolith 0.15.0 makes interactive commands live-extensible, bounds terminal
replay for long conversations, preserves session context through hard recovery,
and makes concurrent child-agent work visible.

## Live commands and self-modification

- Replaced the fixed command table with a layered registry of immutable command
  objects. Help, completion, startup tips, dispatch, busy-turn behavior, and
  terminal ownership now derive from the same live definitions.
- Added `define-application-command` as the supported extension point for user
  initialization and self-modification. Exploratory definitions can be
  installed, discarded, replayed, and made durable without restarting. A
  failed `init.lisp` load restores the exact previous registry.
- Made `/goal` available as a live inspection command while a response is in
  progress, restored saved goals through streamed conversation loading, and
  made command completion query the current registry.
- Replaced reload-sensitive constants with mutable global policy variables.
  Source checks and self-modification now reject `defconstant` and
  `define-constant`, preventing stale compiled values in a live image. The
  constant-specific overlay machinery and Alexandria dependency are gone.

## Long conversations and recovery

- Streamed append-only conversation records instead of materializing complete
  logs in memory, reducing peak memory use for long sessions. Concurrent
  appends are serialized and provider projection keeps a constant-time tail.
- Limited startup and resume presentation to the newest 500 visible transcript
  entries. The new `/history` command pages backward through earlier entries in
  groups of 500 without changing durable history, provider context, or
  compaction.
- Added a per-launch recovery-session handoff so a hard active-image failure
  preserves the conversation identifier and transcript position even when no
  crash capsule can be completed. Recovery also resolves migrated legacy
  identifiers, preserves explicit history pagination, and continues retained
  generation fallback after a configuration-status exit.

## Concurrent tasks

- Added a live child-agent strip above the modeline. It shows queued and running
  jobs in stable order with role, detached state, current activity, and a
  coalesced spinner, while bounding visible rows and summarizing overflow.
- Kept task updates on the input renderer so worker threads never interleave
  terminal control sequences. Finished jobs disappear; the live projection
  itself never enters durable conversation history.
- Clarified for the model that detached tasks can run concurrently while it
  continues useful work, and that it should retain job identifiers, poll or
  wait through the `job.*` tools, and integrate required results before
  finishing.

## Terminal behavior

- Removed transient activity verbs from the modeline.
- Updated Clinedi for stronger repeated Ctrl-C handling during active work and
  shutdown.

## Compatibility notes

- User initialization, overlays, or self-modifications that define constants
  must use `defparameter` for reloadable policy or `defvar` for state intended
  to survive reload.
- Custom command extensions should use `define-application-command`; direct
  modification of the former command table is no longer supported.
- Conversation storage is unchanged. The 500-entry limit applies only to
  terminal replay, and `/history` can display every older page. Full provider
  projection still depends on normal compaction for very long contexts.
- Existing preference files remain readable, including from retained
  generations.
