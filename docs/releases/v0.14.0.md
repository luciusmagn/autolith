# Autolith 0.14.0

Autolith 0.14.0 adds project-scoped adaptation notes, human-friendly
conversation identifiers, and attended release updates, while tightening
terminal shutdown and presentation.

## Project adaptation

- Added an optional root `AUTOLITH.org` ledger for documented project-specific
  adaptations. Its bounded contents supplement `AGENTS.md` as request-local
  context and never enter durable conversation history or compaction.
- Command-line `autolith resume` may offer to create the ledger after a
  qualifying long-running session or repeated substantial project sessions.
  Choosing `not-now` defers the offer for five days, while `never` suppresses
  it for that canonical project path.
- Existing ledgers add a restrained reminder to consider small, evidence-based
  Autolith improvements and keep their scope and verification documented. The
  file is treated as untrusted, non-executable guidance.

## Conversations

- Replaced UUID-style conversation identifiers with seven-character,
  case-sensitive Bitcoin Base58 identifiers displayed in a compact form such
  as `K-8vQ2mp`. Resume accepts both the displayed and unhyphenated forms.
- Added an automatic, restart-safe migration for legacy conversations and
  their durable references and artifacts. Existing UUID resume commands remain
  valid aliases after migration.

## Updates and installation

- Added installation-aware release checks for source, Nix, and selected
  binary-release starts. Source checkouts do not contact the release service,
  and Nix installations receive advice without being modified.
- Selected binary releases now show cached update availability below the
  startup banner and offer `Not now`, `Update now`, and `Skip this version`. A
  bounded background refresh updates only the readable cache and never installs
  unattended.
- Accepted updates restore the terminal before the outer launcher verifies and
  installs the selected release, preserves custom installation roots, and
  restarts with the original arguments.

## Terminal experience

- Simplified the activity row by removing redundant model and effort labels,
  and made reasoning effort red for quicker visual scanning.
- Kept repeated Ctrl-C available throughout graceful shutdown, including
  blocked runtime cleanup, so it can still restore the terminal, print the
  exact resume command for durable conversations, and force exit with status
  130.

## Compatibility notes

- The first startup with legacy UUID conversations performs the identifier
  migration automatically. The migration is resumable after interruption, and
  old identifiers remain resolvable.
- `AUTOLITH.org` is optional. Existing repositories and in-session `/resume`
  behavior are unchanged when it is absent.
