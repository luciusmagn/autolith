# Autolith 0.16.1

Autolith 0.16.1 hardens long-running conversations and makes the release host
update itself from immutable Git tags.

## Conversation reliability

- Added exclusive process-lifetime conversation leases so a recovered process
  and its stale predecessor cannot append to the same history concurrently.
- Claimed resumed conversations before replay or crash repair and preserved
  lease ownership correctly across conversation switches and shutdown.
- Added narrow compatibility handling for a stale tool result written after an
  exact recovery sentinel while continuing to reject other duplicate results.
- Avoided new-conversation identifier collisions by probing the durable store
  before selecting an identifier.

## Bounded tools and provider recovery

- Reworked `fs.read` to stream fixed-size chunks, cap the complete rendered
  result, and count lines without loading an entire large or malformed file
  into the active image.
- Retained exact line-window and truncation reporting for long lines, long
  paths, sparse files, and malformed UTF-8 input.
- Added bounded cancellable retries for structured provider overload and
  slowdown responses and for transient HTTP 500, 502, 503, and 504 failures.

## Autonomous release host

- Added a Common Lisp release-host updater that polls direct semantic Git tags,
  validates their exact commit and declared version, and prepares each checkout
  at its final versioned pathname.
- Runs bootstrap, repository checks, and a fresh candidate probe before
  activation, prepares the private service runtime under its real account,
  then makes deployments root-owned and immutable.
- Added durable native S-expression deployment transactions, atomic current and
  previous selections, exact source-specific health checks, synchronous s6
  activation, rollback, crash reconciliation, and bounded deployment retention.
- Serialized host updates with release builds and added an initial fixed-source
  fallback so 0.16.0 hosts can perform their one attended migration and update
  autonomously afterward.
