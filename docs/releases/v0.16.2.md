# Autolith 0.16.2

Autolith 0.16.2 prevents transient provider transport failures and stale
retained generations from turning recoverable interruptions into fatal or
misleading recovery sessions.

## Provider transport reliability

- Normalized low-level DNS, socket, and TLS failures at the provider boundary
  instead of allowing dependency-specific conditions to terminate Autolith.
- Made transient connection and TLS syscall failures eligible for the existing
  bounded reconnection policy while keeping permanent TLS setup failures typed
  and visible.
- Closed response streams abortively under a non-signaling cleanup boundary so
  cleanup cannot replace either a completed response or its primary stream
  failure.
- Added regression coverage using the exact `CL+SSL::SSL-ERROR-SYSCALL`
  condition observed in long-running sessions.

## Revision-safe recovery

- Restricted automatic crash recovery to retained generations built from the
  currently checked-out Git revision.
- Booted clean current source when no compatible same-revision generation is
  available instead of silently falling back across many releases.
- Preserved deliberate cross-revision recovery through `/rollback`, manual
  recovery, and explicit `--generation` selection.
- Covered initial selection, fallback candidates, fatal status propagation, and
  rollback exceptions through the recovery entry boundary.
