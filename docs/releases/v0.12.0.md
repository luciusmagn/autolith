# Autolith 0.12.0

Autolith 0.12.0 adds native child-agent orchestration, first-class image input,
request-local context extensions, and a dedicated host-built binary release
path.

## Child agents and long-running work

- Added `task.run` for one child assignment or a concurrent batch. Children may
  run synchronously or continue as detached jobs.
- Added `task.agents` plus the bundled `scout`, `designer`, `reviewer`,
  `librarian`, `task`, and `sonic` roles. Project and user roles can override
  them from `.autolith/agents/` and
  `${XDG_CONFIG_HOME:-~/.config}/autolith/agents/` with strictly validated
  `.sexp` definitions.
- Added `job.list`, `job.get`, `job.wait`, and `job.cancel` for observing and
  controlling work. Cancellation cascades through child task trees.
- Kept native task state in readable Lisp data. Role definitions, tool results,
  and retained `result.sexp` artifacts use s-expressions; JSON remains at
  provider and API compatibility boundaries.
- Added bounded pagination to role and job listings, private child transcripts,
  conversation-scoped visibility, structured yields, deadline handling, and a
  bounded scheduler that supports nested delegation without deadlock.
- Child sessions use the ordinary provider, tools, authorization, and compaction
  path with restricted child-safe capabilities and no `self.*` access. Every
  child completes through one explicit `yield.submit` result.
- Removed the former provider-step and 256-tool-call ceilings. A turn now runs
  until completion, interruption, or an unrecoverable failure, with context
  compaction handling long sessions.

## Images and terminal experience

- Added native PNG, JPEG, GIF, and WebP input. Paste or drag image paths into
  the composer, or preload them with `-i FILE` and `--image FILE`.
- Added `fs.view-image` so the model can inspect images created or found during
  a turn without OCR or an ASCII approximation.
- Added persistent `/compact on|off` presentation. Compact mode hides routine
  successful tool output while preserving failures, file writes and edits, and
  arbitrary code execution results.
- Added `/agenda`, grouped `/resume` results for the current workspace, and
  newest-first ordering within each group.
- Extended multiline input with Shift-Enter and Ctrl-Enter alongside Alt-Enter,
  and added ColorLisp edit-diff highlighting.
- Bundled syntax support for Common Lisp, Scheme, Rust, C and C++, Python, Go,
  shell, TOML, JavaScript, TypeScript, TSX, JSON, YAML, Markdown, HTML, CSS, Nix,
  Java, Ruby, and Lua.
- Updated the startup warning to state that sandboxing is no substitute for
  human oversight.

## Context, memory, and customization

- Added `${XDG_CONFIG_HOME:-~/.config}/autolith/init.lisp` as the final
  user-controlled Common Lisp configuration layer.
- Added a request-local context contributor registry shared by tracked code,
  self-modifications, and user Lisp. Contributions stack by default, remain
  outside durable history, and are observable through `/context`.
- Changed memory recall to rank relevant workspace and global memories at the
  provider boundary instead of including the full eligible catalog in every
  request.
- Added stable memory links to agenda items. Replacing a memory preserves its
  agenda attachments.
- Extended `self.source` to inspect exact source from pinned direct ASDF
  dependencies as well as Autolith and its matching SBCL tree.
- Added the `none` reasoning effort and corrected GPT-5.6 context windows used
  for automatic compaction.

## Installation, packaging, and recovery

- Moved binary release delivery to a dedicated Common Lisp service at
  `sh.lambda-symbolics.com`. It fully checks tagged Linux archives on the
  installer host, verifies checksums, and publishes them atomically. The install
  command remains `curl -fsSL https://sh.lambda-symbolics.com/autolith | sh`.
- Fixed `script/bootstrap` so Qlot always resolves the Autolith checkout instead
  of accidentally scanning the caller's home directory and unrelated ASDF
  systems.
- Repaired the Nix flake and added a real startup smoke check, including image
  support, the packaged fff search library, and ColorLisp syntax support.
- Source launches now offer to bootstrap a missing or stale fast-start image.
  `autolith --from-source` deliberately bypasses that prompt and image checks.
- Recovery now prints a red `RECOVERY IMAGE` explanation, colors the recovered
  AL banner red, and validates private image commits before advancing their
  selection.
- Hardened private state permissions, atomic publication, search and task
  runtime replacement, and packaged Git maintenance.
- Extracted reusable readable persistence, indexed search, and isolated Lisp
  worker mechanics into the pinned `sexp-store`, `clifff`, and `sbcl-workers`
  libraries without changing Autolith's native data formats.

## Compatibility notes

- Use `autolith resume [CONVERSATION]`. The duplicate `--resume` spelling has
  been removed.
- `/compact` without an argument still performs immediate context compaction;
  only `/compact on|off` changes the saved presentation preference.
- GitHub releases now carry metadata and these curated notes. Binary archives
  and checksums are published by `sh.lambda-symbolics.com`.
