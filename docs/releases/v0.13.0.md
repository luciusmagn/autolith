# Autolith 0.13.0

Autolith 0.13.0 improves long-running reliability and terminal presentation
with resilient provider retries, bounded child-agent runtimes, highlighted code
blocks, a contextual activity row, and startup command tips.

## Long-running reliability

- Added bounded retries for provider streams that close, truncate an SSE event,
  or otherwise end before a valid terminal response. Retry progress remains
  visible in the terminal.
- Added bounded retries for transient structured `server_error` and
  `internal_server_error` events while preserving immediate failure for
  permanent provider errors.
- Persisted terminal provider failure details outside model replay so failed
  requests remain diagnosable after restart without polluting conversation
  context.
- Gave concurrent child agents distinct provider conversation identities.
- Added a 30-minute default wall-clock limit for child agents so stalled work
  cannot remain alive indefinitely. Set `AUTOLITH_TASK_MAX_RUNTIME_MS=0` to
  restore uncapped child runtimes.
- Updated the native indexed-search worker so ordinary watched-file changes
  keep the same helper process alive.

## Terminal experience

- Added syntax highlighting to fenced Markdown code blocks using the same
  ColorLisp language support as edit diffs.
- Adopted `cl-colorist` as the shared terminal styling layer for Autolith and
  Clinedi.
- Added a full-width activity row with a neutral background, a Lisp-themed
  spinner, elapsed and stale-progress timing, and colored model, effort, and Git
  branch details.
- Added one colored interactive-command tip beneath each startup banner.
  Command definitions now require a non-empty tip at compile time, and aliases
  share their canonical command's tip.

## Installation and packaging

- Repaired the tagged Ubuntu release builder so ColorLisp compiles in its
  pinned environment and older unpublished tags can complete.
- Updated ColorLisp to resolve Tree-sitter Unicode headers from its vendored
  source instead of relying on host ICU headers.

## Compatibility notes

- Child agents now time out after 30 minutes by default. Configure
  `AUTOLITH_TASK_MAX_RUNTIME_MS` when a different limit is appropriate.
- No interactive command spellings or persisted data formats changed in this
  release.
